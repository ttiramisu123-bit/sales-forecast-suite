**核心痛点**：
1. 几乎所有面板绑定 `selectedMonth`，看 12 个月全貌要切换 12 次
2. SKU 调整默认只影响单月，运营常态是"全月生效"
3. SI / Type 目标 / SKU 调整全手动逐行输入，无批量导入
4. SI 线和销量共用一个 Y 轴，SI 被压扁
5. 历史数据和预测数据之间无视觉分界

### #1 12月目标矩阵面板

**现状**：Type 目标表只显示 `selectedMonth` 的数据，看 12 个月要切换 12 次。

**改成**：一张横向表格，12 个月全部展开。

```

┌──────────────┬───────┬───────┬───────┬───────┬───────┬───────┬───┬───────┐

│ 📝 备注      │   —   │   —   │   —   │   —   │ 黑五   │ 圣诞   │   │   —   │  ← 12个文字框
├──────────────┼───────┼───────┼───────┼───────┼───────┼───────┼───┼───────┤

│              │       │       │       │       │建议18.3%│     │   │       │  ← 灰色小字=建议值

└──────────────┴───────┴───────┴───────┴───────┴───────┴───────┴───┴───────┘
```

**交互规则**：

| 格子状态 | 显示 | 行为 |
|----------|------|------|
| 未手动改过 | 显示建议值（如 7.6%） | 全局目标变了 → 自动重算更新 |
| 手动改了 | 输入框里是手动值，下方灰色小字 `建议18.3%` | 全局目标变了 → 手动值不动，灰色小字更新 |

**实现要点**：
- 去 `selectedMonth` 参数，`typeTargetSuggestionRows()` 接收 `globalRate` 参数
- 全局目标行：`state.promoCalendar.monthlyRates` 驱动
- 备注行：`state.promoCalendar.notes` 驱动
- 每个 Type 的格子：`state.adjustments.typeTargetOverride.get("Fuel Pumps|2026-07")` 判断是否手动覆盖
- 顶部保留一个「基线涨幅」输入框，改它 → 12 格全刷新（被手动覆盖的格不动）
- `applyTarget()` → 循环 12 个月，每月用对应 globalRate

**涉及文件**：`index.html`（改 30 行）、`app.js`（改 `renderTypeTargetPanel` + `typeTargetSuggestionRows` + `applyTarget`，约 60 行）

### #8 SI 双 Y 轴

**现状**：SI 曲线（0.5-1.8）和销量曲线（0-10000+）共用一个 Y 轴，SI 线被压成紧贴底部的直线。

**改成**：左侧 Y 轴 = 销量/销售额，右侧 Y 轴 = SI（范围 0.3-2.0）。

```
        销量（左轴）                  SI（右轴）
10000 │                                                              │ 2.0
      │  ●───●───●───●───●───●───●───●───●───●───●───●              │
 5000 │                                     ●───●───●───●───●         │ 1.5
      │ · · · · · · · · · · · · · · · · · · · · · · · · · · · · · · │ (虚线=历史/预测分界)
    0 │___________________________________________________________ │ 1.0
        7月  8月  9月 10月 11月 12月  1月  2月  3月  4月  5月  6月
        ← 历史─────────│─────────预测 →
```

**实现要点**：
- `drawTrend()` 里新增右侧 Y 轴绘制
- `ySi()` 函数返回 `ySi(value) = chartBottom + (1 - (value - 0.3) / 1.7) * chartHeight`
- 图例中 SI 曲线上标注「(右轴)」

**涉及文件**：`app.js`（改 `drawTrend` 约 30 行）

**功能流程**：

```
选模板 → 下载 CSV → Excel 编辑 → 上传
  ↓
扫描表头 → 自动识别模板类型
  ├─ 含"月份"+"全局涨幅" → 模板2b（月度全局目标）

  ↓
```

### #25 CSV 导入 SI（季节系数）

**驱动关系**：SI 直接影响公式预测。目前几十个 Type × 12 月全在 SI 编辑器里逐行手改，太慢。

**CSV 格式**：

```
Type,1月,2月,3月,4月,5月,6月,7月,8月,9月,10月,11月,12月
Fuel Hose,0.88,0.95,1.42,1.18,1.04,0.98,0.94,0.98,1.05,1.12,1.18,1.08
Fuel Pump,0.94,0.8,0.86,0.83,0.9,0.94,0.98,1.08,1.17,1.17,1.17,1.17
Rail Kit,1.22,1.12,1.08,1.07,1.06,1.04,1.01,0.93,0.9,0.88,0.88,0.97
```

**列说明**：

| 列 | 必填 | 类型 | 说明 |
|-----|------|------|------|
| `Type` | ✅ | 文字 | 类目名称，必须精确匹配 |
| `1月`~`12月` | ❌ | 数字 | SI 值，范围 0.3-2.5。允许留空，留空=不覆盖该月 |

**校验规则**：
- Type 不存在 → 跳过并记录
- 值超出 0.3-2.5 → 警告（自动 clamp）
- 空值 → 不修改该月

**导入后写入**：
```js
state.si["Fuel Hose"] = [0.88, 0.95, 1.42, 1.18, 1.04, 0.98, 0.94, 0.98, 1.05, 1.12, 1.18, 1.08]
```

**模板生成**：`downloadSiTemplate()` — 如果 `state.si` 已有数据则预填；新建则 Type 列列出当前导入数据中所有 Type，SI 列全填 1.0。

### #26 CSV 导入 Type 目标覆盖

**驱动关系**：覆盖摊分逻辑生成的建议值。留空的单元格 = 使用建议值。

**CSV 格式**：

```
Type,2026-07,2026-08,2026-09,2026-10,2026-11,2026-12,2027-01,2027-02,2027-03,2027-04,2027-05,2027-06
Fuel Pumps,12,,, ,15,8,,,,,,,
Fuel Filters,10,,, ,18,8,,,,,,,
```

**列说明**：

| 列 | 必填 | 类型 | 说明 |
|-----|------|------|------|

### #27 CSV 导入 SKU 调整

**驱动关系**：覆盖系数 (skuFactor) / 加量 (eventAdd) / 直改 (directUnits)。三合一模板，每行至少填一种调整。

**CSV 格式**：

```
SKU,月份,系数,加量,直改,备注
FUEL-PUMP-023,2026-07,1.2,,,Prime Day促销备货
FUEL-PUMP-045,2026-07,,50,,活动加量
FUEL-PUMP-067,2026-07,,,800,供应商锁定直改
FUEL-PUMP-023,2026-11,1.5,,,黑五加码
FUEL-FILTER-012,2026-07,0.8,,,淡季压系数
FUEL-HOSE-003,2026-07,,,-50,清仓直改为0
```

**列说明**：

| 列 | 必填 | 类型 | 说明 |
|-----|------|------|------|
| `SKU` | ✅ | 文字 | 产品编码，必须精确匹配数据中的 SKU |
| `月份` | ✅ | YYYY-MM | 预测月份 |
| `系数` | ❌ | 数字 | 策略系数，1.0 = 不变。范围 0.1-5.0 |
| `加量` | ❌ | 整数 | 活动加量，0 = 不加。可为负数 |
| `直改` | ❌ | 整数 | 直接锁定销量。填了会忽略公式预测。可为 0（表示清零） |
| `备注` | ❌ | 文字 | 手动标记，存到 `state.adjustments.notes` |

**校验规则**：

| 检查 | 结果 |
|------|------|
| SKU 不存在 | 跳过 + 记录 |
| 月份不在预测范围 | 跳过 + 记录 |
| 系数 < 0.1 或 > 5 | 警告 + clamp |
| 「系数」「加量」「直改」全空 | 跳过（此行无效） |
| 「直改」和「系数」同时填 | 提示"直改会覆盖系数效果" |

**导入后写入**：
```js
// 行1: FUEL-PUMP-023, 2026-07, 系数=1.2
state.adjustments.skuFactor.set("FUEL-PUMP-023|2026-07", 1.2)
state.adjustments.notes.set("FUEL-PUMP-023|2026-07", "Prime Day促销备货")

// 行2: FUEL-PUMP-045, 2026-07, 加量=50
state.adjustments.eventAdd.set("FUEL-PUMP-045|2026-07", 50)

// 行3: FUEL-PUMP-067, 2026-07, 直改=800
state.adjustments.directUnits.set("FUEL-PUMP-067|2026-07", 800)
```

**模板生成**：`downloadSkuAdjustTemplate()` — 表头行 + 一行示例数据（带注释说明格式）。

## 6. CSV 导入模板规范

### 通用规则

- 所有模板编码：`UTF-8 with BOM`（Excel 兼容）
- 表头匹配：**大小写不敏感**，`SKU`/`sku`/`Sku` 均可识别
- 空值：CSV 中连续逗号 `,,` 表示跳过该列，不覆盖当前值
- 日期格式：统一 `YYYY-MM`（如 `2026-07`）
- 导入前自动 `trim()` 所有单元格
- 导入后 **自动 pushHistory()**，支持 Ctrl+Z 回退整批
