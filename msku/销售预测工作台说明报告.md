# EVIL ENERGY 销售预测工作台说明报告

版本：v1  
适用范围：SKU 销量预测、Type/类目销售额趋势、月度目标分配、云端版本协作  
核心定位：用网页工作台替代 Excel 手工公式，统一预测口径、减少误操作，并支持多人查看最新云端版本。

## 一、表格/网页用途

本工作台用于基于历史销量、SKU 价格、Type/类目、SI 季节性指数和人工运营调整，自动生成未来 12 个月的 SKU 预测销量、预测销售额，以及 Type/类目维度销售额汇总趋势。

系统会自动按“历史销量最后一个月份的下个月”开始预测。例如历史销量最后一个月是 2026-06，则预测月份为 2026-07 至 2027-06。

## 二、导入数据口径

### 1. 基础销量模板

必填字段：

| 字段 | 说明 | 影响 |
|---|---|---|
| SKU | SKU 编码，作为预测最小颗粒度 | 决定 SKU 明细、人工调整、历史预测匹配 |
| 价格 | SKU 单价 | 决定销售额汇总，销售额 = 销量 × 价格 |
| Type/类目 | SKU 所属类目/品线 | 决定 SI 匹配、Type 汇总、目标涨幅分配 |
| 历史月份销量 | 按月历史销量，如 2025/7/1、2025-07 | 决定清洗、趋势、稳定度、预测基准 |

可选字段：

| 字段 | 说明 |
|---|---|
| Status | SKU 状态，仅作为参考字段 |
| 历史预测版本 | 过去某次做好的未来预测值，只用于曲线对比和参考，不直接参与公式预测 |

### 2. SI 季节性指数

SI 指数用于表达汽配类目不同月份的旺淡季。系统内置默认 SI，也支持在工作台内维护各 Type 的 12 个月 SI。

匹配顺序：

1. Type 精确匹配 SI。
2. Type 名称标准化后智能匹配。
3. 找不到时使用“全部”默认 SI。

若某个 Type 没有 SI，系统会提示缺失，并允许在 SI 指数维护区域补充。

## 三、整体预测链路

预测链路分为 6 层：

| 层级 | 名称 | 作用 |
|---|---|---|
| 1 | 数据清洗 | 去除异常高值、异常归零值 |
| 2 | 稳定度识别 | 判断 SKU 属于高稳定、中稳定、低稳定、样本不足 |
| 3 | 趋势均值 | 用清洗后历史销量计算预测均值 |
| 4 | 季节性修正 | 按 Type SI 对月份进行旺淡季调整 |
| 5 | 增长限制 | 防止单月预测跳涨或跳跌过大 |
| 6 | 人工调整 | 支持 Type 目标、SKU 系数、活动加量、SKU 直改 |

最终输出分为两类：

| 输出 | 说明 |
|---|---|
| 公式预测 | 系统根据历史销量、趋势、稳定度、SI 自动生成 |
| 最终预测 | 公式预测叠加人工调整后的结果，是运营最终使用口径 |

## 四、核心公式口径

### 1. 数据清洗

系统先把历史销量转为非负数：

```text
有效销量 = MAX(0, 原始销量)
```

异常值判断包含两类：

| 类型 | 判断逻辑 | 替换方式 |
|---|---|---|
| 异常高值 | 当前值同时高于历史基准倍数、邻近月份倍数 | 用左右邻近月份均值替换 |
| 异常归零 | 当前为 0，但邻近月份均值明显不低 | 用左右邻近月份均值替换 |

清洗后的销量会用于后续稳定度、趋势、均值和预测计算。异常月份会在诊断信息中标出。

### 2. 稳定度判断

稳定度基于清洗后销量的活跃月份和波动系数 CV：

```text
CV = 标准差 / 正销量均值
```

| 稳定层级 | 判断口径 | 预测特点 |
|---|---|---|
| 高稳定 | CV 较低 | 更相信 SKU 自身历史，预测更稳 |
| 中稳定 | CV 中等 | SKU 趋势和类目趋势均衡 |
| 低稳定 | CV 较高 | 更依赖 Type/类目趋势，避免 SKU 历史噪音放大 |
| 样本不足 | 有销量月份少于 3 个月 | 保守预测 |

### 3. 趋势计算

SKU 趋势：

```text
SKU趋势 = 最近窗口销量均值 / 前一窗口销量均值
```

系统会限制趋势范围，避免历史短期波动导致预测过度放大：

```text
SKU趋势限制范围：0.90 至 1.12
Type趋势限制范围：0.90 至 1.12
```

Type 趋势是同一 Type 下所有 SKU 历史销量汇总后计算。

### 4. 趋势均值

```text
清洗后正销量均值 = 清洗后销量中大于 0 的月份均值
综合趋势 = SKU趋势 × SKU权重 + Type趋势 × Type权重
趋势均值 = 清洗后正销量均值 × 综合趋势
```

稳定度不同，SKU 权重和 Type 权重不同。越稳定的 SKU 越看自身历史，越不稳定的 SKU 越跟随 Type 趋势。

### 5. SI 季节性修正

```text
季节性因子 = 1 + (SI - 1) × 季节权重
初始预测销量 = 趋势均值 × 季节性因子
```

说明：

- SI > 1 表示旺季月份。
- SI < 1 表示淡季月份。
- 季节权重由稳定度决定，低稳定 SKU 不会完全照搬历史单 SKU 波动，而是更受 Type 趋势约束。

### 6. 月增长限制

为防止预测销量单月跳变过大，系统会同时参考环比上限和历史月增长数量。

当预测值高于上月预测值时：

```text
环比上限 = 上月预测 × (1 + 上涨限制)
数量上限 = 上月预测 + 历史平均正增长量 × 增长系数
最终公式预测 = MIN(初始预测, 环比上限, 数量上限)
```

当预测值低于上月预测值时：

```text
最终公式预测 = MAX(初始预测, 上月预测 × (1 - 下跌限制))
```

旺季月份会允许更高增长空间，但仍会被上限约束。

## 五、最终预测公式

最终预测以公式预测为基准，再叠加人工调整。

### 1. 普通情况

```text
最终预测销量 =
公式预测销量
× SKU系数
× Type目标系数
× SKU稳定度微调系数
+ 活动/清仓加量
```

最终取整并不低于 0：

```text
最终预测销量 = MAX(0, ROUND(最终预测销量))
```

### 2. SKU 直改优先

如果运营对某个 SKU 某个月做了“直改最终销量”，则优先级最高：

```text
最终预测销量 = SKU直改值
```

此时 SKU 系数、Type 目标、活动加量不再覆盖该单元。

### 3. 销售额

```text
公式预测销售额 = 公式预测销量 × SKU价格
最终预测销售额 = 最终预测销量 × SKU价格
历史预测销售额 = 历史预测销量 × SKU价格
```

销售额汇总以 Type/类目维度和月份维度进行加总。

## 六、月度目标分配逻辑

月度目标分配用于把运营填写的全局增长目标，合理分配到各 Type。

### 1. Type 调整倾向系数

```text
T = 稳定度因子 × 趋势因子
```

稳定度因子：

| 稳定度 | 因子 |
|---|---:|
| 高稳定 | 0.8 |
| 中稳定 | 1.0 |
| 低稳定 | 1.3 |
| 样本不足 | 0.6 |

趋势因子基于 Type 最近 3 个月均值 / 前 3 个月均值：

| 趋势状态 | 判断 | 因子 |
|---|---|---:|
| 上升 | > 1.15 | 1.3 |
| 平稳 | 1.00 至 1.15 | 1.0 |
| 微降 | 0.85 至 1.00 | 0.8 |
| 下降 | < 0.85 | 0.5 |

### 2. 全局涨幅分配到 Type

```text
Type建议涨幅 = 全局目标涨幅 × (Type T / 所有 Type 的 T 均值)
```

为防止单个 Type 被推得过高或过低，系统设置封顶兜底：

```text
Type建议涨幅限制范围：-5% 至 +25%
Type目标系数限制范围：0.3 至 2.0
```

### 3. SKU 稳定度微调

Type 涨幅应用到 SKU 后，会做轻量 SKU 级微调：

```text
SKU微调因子 = 1 + 0.15 × (SKU稳定度因子 - Type加权稳定度因子)
```

并限制在：

```text
0.93 至 1.07
```

为了保证 Type 总量不被微调显著改变，系统会做加权归一：

```text
最终SKU微调 = SKU微调因子 / Type内加权平均微调因子
```

这意味着：

- 高稳定 SKU 不会被过度加量。
- 低稳定 SKU 会获得一点弹性。
- Type 总目标基本保持不变。

## 七、人工调整优先级

从高到低：

| 优先级 | 调整方式 | 说明 |
|---:|---|---|
| 1 | SKU 直改 | 直接指定某 SKU 某月最终销量，优先级最高 |
| 2 | 活动/清仓加量 | 在公式预测基础上增加固定销量 |
| 3 | SKU 系数 | 对某 SKU 某月乘以系数 |
| 4 | Type 目标系数 | 对某 Type 某月批量调整 |
| 5 | 公式预测 | 默认基础预测 |

## 八、页面板块构成

### 1. 导入数据

用于导入基础销量模板、历史预测版本、项目 JSON。

### 2. 状态与辅助

用于选择当前月份、辅助搜索 SKU/Type、撤销和重做。

### 3. 批量导入调整

支持导入：

- 月度全局目标
- Type 目标覆盖
- SI 指数
- SKU 调整项

### 4. 保存与导出

支持：

- 保存项目 JSON
- 导出历史 + 预测 Excel
- 导出只读 HTML 报告
- 下载导入模板

### 5. 云端同步

云端地址已内置，运营无需填写 URL。

支持：

- 自动导入云端最新版本
- 保存当前版本到云端
- 查看云端版本

### 6. KPI 指标卡

展示当前月份：

- SKU 数
- 公式预测销量
- 最终预测销量
- 最终预测销售额
- 人工修正 SKU 数
- SI 缺失类目数

### 7. 月度销售额趋势

展示未来 12 个月最终预测销售额柱状趋势，并显示环比变化。

### 8. 月度目标分配

用于填写全局目标涨幅，生成 Type 建议涨幅，再应用到 SKU。

操作顺序：

1. 填写某月全局目标涨幅。
2. 点击“全局应用”，生成 Type 建议。
3. 如有需要，手动修改某个 Type 的涨幅。
4. 点击“类目应用”，写入 SKU 最终预测。

### 9. Type/类目销售额汇总趋势

按当前月份汇总每个 Type 的公式销量、最终销量、最终销售额、占比和环比。

### 10. SI 指数维护

维护 Type 的 12 个月 SI 指数。导入数据后如果发现新 Type 没有 SI，可在这里补充。

### 11. 筛选区

SKU 和 Type 共用筛选栏，筛选结果会同步影响：

- 趋势曲线
- SKU 预测明细

### 12. 趋势曲线

展示：

- 历史销量
- 公式预测
- 最终预测
- 历史预测
- SI 曲线

支持 hover 查看具体月份数值，并可通过图例开关曲线。

### 13. SKU 预测明细

展示 SKU 级别预测数据，支持分页、筛选和单 SKU 调整。

## 九、云端版本规则

云端保存由 Google Apps Script + Google Sheet + Google Drive 组成：

| 位置 | 保存内容 |
|---|---|
| Google Sheet | 版本索引、保存人、备注、Drive 文件 ID、是否最新、是否长期保留 |
| Google Drive | 完整项目 JSON |
| GitHub Pages | 网页代码，不保存预测数据 |

版本保留规则：

| 类型 | 规则 |
|---|---|
| 普通版本 | 只保留最新 10 份 |
| 正式版/月度定版 | 长期保留 |
| 超出数量的旧普通版本 | Drive JSON 移到垃圾箱，Sheet 记录删除 |

长期保留判断：

```text
版本备注包含“正式版”或“月度定版”时，keep_forever = TRUE
```

## 十、运营标准操作流程

### 日常查看

1. 打开网页。
2. 系统自动导入云端最新版本。
3. 查看 KPI、月度销售额趋势、Type 汇总和 SKU 明细。

### 日常调整

1. 先确认当前云端版本是否最新。
2. 修改月度目标或 SKU 调整。
3. 点击相应应用按钮，使调整写入最终预测。
4. 查看 KPI、Type 汇总和趋势曲线是否合理。
5. 填写保存人和版本备注。
6. 点击“保存云端”。

### 正式定版

版本备注建议写：

```text
2026年7月正式版
```

或：

```text
2026年7月月度定版
```

这样系统会自动长期保留，不受普通版本 10 份限制影响。

## 十一、注意事项

1. 价格为空或为 0，会导致销售额偏低或为 0。
2. Type 名称不统一，会影响 SI 匹配和 Type 汇总。
3. SKU 直改优先级最高，若发现某 SKU 不随 Type 目标变化，先检查是否存在直改。
4. 保存云端前建议填写保存人和备注，方便追溯。
5. 若多人同时编辑，建议先导入最新版本再调整，避免覆盖别人刚保存的方案。
6. 若页面没有自动加载最新版本，可手动点击“导入最新”。

## 十二、口径总结

本工作台的核心预测口径是：

```text
先清洗历史销量
再计算 SKU 稳定度和 Type 趋势
再结合 SI 季节性生成公式预测
再通过 Type 目标、SKU 调整、活动加量和直改形成最终预测
最后按 SKU 价格汇总销售额
```

其中：

- 公式预测用于提供系统基准。
- 最终预测用于运营决策和销售额汇总。
- 历史预测只用于曲线对比，不参与当前公式计算。
- 云端最新版本作为团队协作的统一查看口径。
