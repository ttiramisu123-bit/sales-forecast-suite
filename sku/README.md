# EVIL ENERGY SKU 需求预测工作台

这是独立的 SKU 维度预测系统，和 MSKU 预测系统分开运行。页面初始化只轻量读取内置映射/状态文件；预测计算、迭代替换、来源追溯、云端导入都在用户触发后才执行。

## 输入

1. MSKU 预测包
   - 可从云端导入最新 `sku_input_package`
   - 也可手动上传 MSKU 预测结果

2. MSKU-SKU 映射表
   - 默认读取 `data/msku_sku_mapping.json`
   - 字段：`MSKU`、`SKU`、`用量`、`SKU名称`、`SKU状态`、`SKU类目`、`是否通用件`、`备注`

3. SKU 销售状态表
   - 默认读取 `data/sku_status.json`
   - 字段：`SKU`、`SKU名称`、`SKU类目`、`销售状态`、`迭代SKU`、`责任人`、`备注`
   - `清库`、`清仓`、`停售`、`下架` 会自动转入 `迭代SKU`
   - 其他状态按正常 SKU 处理，不报错

4. 人工调整表，可选
   - 字段：`SKU`、`月份`、`SKU整体系数`、`单月系数`、`加减量`、`直改`、`备注`

## 三版结果

- 映射版本：`MSKU最终预测销量 × 映射用量`
- 系统平衡版：映射版本按牛鞭风险系数压制，只压制不放大
- 人工终版：系统平衡版再叠加人工系数、单月加量或单月直改

## SKU 迭代处理

如果某个 SKU 状态为清库/清仓/停售/下架：

- 有迭代 SKU：预测需求自动转入迭代 SKU，并与迭代 SKU 自身需求聚合
- 没有迭代 SKU 且存在预测需求：阻止生成，并提示需要补充迭代关系
- 多级迭代支持，例如 `A -> B -> C` 最终聚合到 `C`
- 循环迭代会阻止生成，例如 `A -> B -> A`

来源明细不会预先全量展开。点击某个 SKU 后，才展示该 SKU 的 MSKU 来源、原始 SKU、迭代路径和风险提示。

## 导出

导出的 Excel 包含：

- SKU预测横向表：三版月度需求，带 SKU 状态、转入来源数、责任人
- 风险诊断表：风险分、风险等级、压制系数、风险原因
- SKU来源明细表：MSKU 来源、原始 SKU、最终 SKU、是否转入、迭代路径
- 人工调整表：当前人工调整记录
